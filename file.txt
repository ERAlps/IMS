### 浏览器打不开页面？按以下步骤排查：

1. **查看群晖实时日志 (最有效)**：
   - 打开群晖 **Container Manager**。
   - 点击左侧 **容器**，找到 `smart-ims-web`。
   - 点击该容器，进入详情页。
   - 点击上方的 **日志** 选项卡。
   - 如果 PHP 有报错，这里会显示红色文字。如果这里一片空白，说明请求根本没到 PHP（可能是网络问题）。

2. **网络端口检查**：
   - 确认访问的是 `http://[群晖IP]:8085`。
   - 如果群晖开启了防火墙，请确保 8085 端口已放行。
   - 在 Container Manager 的 `smart-ims-web` 详情页，检查“端口设置”，确认本地端口是 8085，容器端口是 80。

3. **使用调试工具**：
   - 尝试访问 `http://[群晖IP]:8085/debug.php`。
   - 如果这个页面能打开，说明网络和 Apache 没问题，问题出在数据库或 JavaScript。
   - 如果这个页面也打不开（显示“连接已重置”），说明 Docker 容器可能没跑起来，或者端口冲突。

4. **浏览器控制台 (F12)**：
   - 在浏览器按 `F12`，点击 `Console`（控制台）。
   - 查看是否有报错。如果有 `Failed to load resource`，通常是网络不通。
   - 注意：`index.tsx` 使用了 ESM 导入，如果你的网络无法访问 `esm.sh`，页面会一直显示加载中。

5. **最后大招**：
   - 如果还是不行，在 Container Manager -> 项目里，点击 **停止**，然后点击 **操作 -> 清除**（这会清除容器但保留卷），再重新 **启动**。
